# v0.0.7 Release Notes

**Status:** GitHub pre-release published
**Release date:** July 17, 2026
**Compared with:** v0.0.6
**Version code:** 7
**Default APK:** Online TTS release

## Highlights

- Added a Top scenario acceptance list and a structured failure sample library for release regression and follow-up tracking.
- Improved Agent result presentation so model errors, provider failures, HTML error pages, internal JSON, and protocol fragments are not shown directly to users.
- Improved approval cards and tool execution state tracking, including accepted, rejected, cancelled, interrupted, completed, and unknown-outcome states.
- Added task-list based Agent execution guards to make multi-step tasks less repetitive and easier to finalize.
- Improved camera, screenshot, cross-app page reading, calendar creation, feedback submission, image summary, and model configuration reliability.
- Added the foundation for local L1 personal memory with memory types, candidate extraction, policy filtering, Room storage, prompt injection, migration, and a User Profile settings page.
- Improved privacy protection with sensitive data redaction, safer diagnostic log formatting, backup exclusions for memory and Agent data, and HTTP privacy tests.

## Agent Result and Tool State Reliability

- Model and provider failures are classified before they are shown to the user.
- Home display, copy, share, and TTS use the same sanitized presentation text.
- Tool calls persist explicit running and terminal states so cancelled, rejected, interrupted, completed, and unknown outcomes can be recovered.
- Task-list execution helps prevent repeated calls after a task has already completed.

## Local Personal Memory

This version introduces the local L1 personal memory foundation. Users can manage memory from settings, while local policy and safety filtering decide whether extracted candidates are stored. Memory is treated as local context support and does not override current user instructions, permissions, or approval policies.

## Known Limitations

- L1 memory is a local foundation only; broader long-term experiences, future tasks, and habits remain future work.
- Diagnostic log auto-upload was documented as a plan/API boundary, not as a completed silent upload flow.
- Full Top scenario and multi-device regression continued into later versions.
