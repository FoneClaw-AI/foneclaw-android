# v0.0.6 Release Notes

**Status:** Release package prepared
**Release date:** July 9, 2026
**Compared with:** v0.0.5
**Version code:** 6
**Default APK:** Online TTS release

## Highlights

- Added the foundation for official FoneClaw plugins, including marketplace search, install proposals, APK verification, trusted plugin refresh, and dynamic tool registration.
- Added a File Manager Plugin example with listing, search, reading, writing, folder creation, rename, batch rename, delete, and HTTPS download tools.
- Added safer plugin execution through signed APK checks, extension manifests, protected Android services, and existing tool approval policies.
- Added Agent checkpoint and history compaction to keep long conversations more stable as context grows.
- Added Tool Approval Mode with Auto approve, Follow tool policy, and Deny all options.
- Improved approval handling across workflows, shortcuts, notification panel actions, and direct tool paths.
- Improved mail account parsing and IMAP/SMTP timeout handling.

## Plugin Foundation

FoneClaw can now discover official plugin capabilities, show a user-reviewable install proposal, verify the downloaded APK, and register trusted plugin tools into the Agent runtime. Plugin packages are distributed from the plugin repository under `plugin/plugins/`, not from the app release notes directory.

The first official example is the File Manager Plugin. It requires FoneClaw Android `0.0.6` or later and exposes file-management tools after the user grants Android All files access to the plugin APK.

## Agent and Approval Reliability

- Checkpoints and compaction reduce the amount of full conversation history sent back into later Agent requests.
- Tool approval decisions now honor the global approval mode across Agent calls, workflow execution, shortcuts, and direct handlers.
- Unknown or disabled tools are denied instead of being auto-approved.
- Mail account settings now trim common fields and use explicit connection, read, and write timeouts.

## Known Limitations

- This package was prepared during early plugin rollout and should be treated as a pre-release package.
- Plugin trust depends on host-side trusted signing fingerprints; plugin repository metadata alone is not a trust root.
- Long-conversation checkpoints improve prompt size pressure but are not the same as user-controlled long-term memory.
