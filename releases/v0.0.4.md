# v0.0.4 Release Notes

**Status:** Release notes draft
**Target:** TBD
**Compared with:** v0.0.3

## Highlights

- Skill import from pasted `SKILL.md` text or trusted HTTPS install links.
- Skill preview with execution flow, referenced tools, validation status, and current state before enabling.
- User Skill lifecycle: save draft, install, enable, disable, inspect, and remove.
- Android deep links for `foneclaw://skill/import?url=...` and verified HTTPS install links.
- Voice Output settings with online, offline, and system TTS backend variants.
- TTS voice selection with local ZIP import and remote voice catalog download support.
- First-run onboarding guide with localized visual assets.
- New conversation message model for text, attachments, tool calls, approvals, and streaming assistant deltas.
- Localized tool fallback messages, RTL/Markdown table fixes, and image-summary upload fixes.

## Skill Import

- Supports GitHub-safe links such as `https://www.foneclaw.ai/skill/install?url=...`.
- Only HTTPS Skill source URLs are accepted.
- Remote Skill files are size-limited and hashed with SHA-256.
- Only enabled and valid user Skills are injected into the agent prompt.
- Includes `daily-device-brief` as the first public sample Skill.

## Voice Output

- Adds Voice Output controls in Settings.
- Adds play/stop actions for assistant replies.
- Adds online TTS relay, offline Sherpa ONNX TTS, and system TTS variants.
- Adds local voice package import and remote voice package download support.

## Fixes

- Fixes Arabic RTL and Markdown table rendering.
- Fixes localized Settings TTS copy.
- Fixes image summary upload and Home attachment syncing.
- Fixes empty stream-frame colon handling.
- Fixes message overwrite and Home image transition message issues.

## Notes

This version focuses on making FoneClaw more extensible through user Skills and more usable through voice output, onboarding, and conversation reliability work.
