# v0.0.5 Release Notes

**Status:** Release notes draft
**Target:** TBD
**Compared with:** v0.0.4

## Highlights

- Improved Agent replies so tool results, approvals, and image summaries are shown more clearly.
- Reduced exposed JSON, internal placeholders, and reasoning artifacts in chat responses.
- Added smarter handling for screenshots, cross-app screenshots, camera results, calendar, memo, mail, SMS, contacts, Wi-Fi, and workflow results.
- Improved approval cards for sensitive actions with clearer titles, buttons, and target details.
- Fixed feedback submission and screenshot upload to the support center.
- Updated onboarding visuals and added failure samples for better regression tracking.

## Agent Experience

- Cleans up Rich UI JSON artifacts, embedded JSON, dangling code fences, and internal placeholders before displaying assistant replies.
- Uses successful tool results as the display fallback when the model reply is empty, invalid, or not user-readable.
- Improves local result messages for calendar, memo, SMS, web search, mail, contacts, call log, Wi-Fi, workflow, camera, and screenshot tools.
- Keeps returned image summaries aligned with the app or system language.

## Approvals and Screenshots

- Adds clearer approval card titles, subtitles, action labels, and field labels for sensitive actions.
- Adds target app details for cross-app screenshot approvals.
- Adds a cross-app screenshot flow that opens or switches to a target app, waits briefly, and captures the screen.
- Reduces repeated follow-up actions after a cross-app screenshot has already completed.

## Feedback

- Fixes Support Center submission configuration for FoneClaw.
- Fixes screenshot upload signing and response parsing.
- Adds privacy-conscious diagnostic logging for feedback submission and upload failures.

## Notes

This version focuses on making FoneClaw's Agent output cleaner, tool execution easier to understand, and user feedback collection more reliable during early testing.
