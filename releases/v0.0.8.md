# v0.0.8 Release Notes

**Status:** GitHub pre-release published
**Release date:** July 22, 2026
**Compared with:** v0.0.7
**Version code:** 8
**Default APK:** Online TTS release

## Highlights

- Upgraded Koog to 1.1.1 and the Java runtime to 21, including compatibility handling for legacy tool-call arguments and stricter OpenAI-compatible services.
- Improved image and attachment reliability with model vision capability controls, safer URL handling, refreshed temporary image access, and clearer 400/404 guidance.
- Added permission recovery that explains missing access, opens the relevant settings, and can resume the original task after the user returns.
- Fixed repeated send-and-cancel flows so completed cleanup no longer stalls the next Agent request or keeps cancelled content in later prompts.
- Added a modular, privacy-safe Firebase Analytics integration for task, approval, tool, permission, onboarding, Daily Brief, and feedback events.
- Improved image summary retry behavior, camera and screenshot recovery, approval binding, tool policies, and localized failure presentation.

## Agent Runtime and Media Reliability

- Tool-call compatibility was hardened for stricter OpenAI-compatible providers.
- Current image attachments are handled more safely, while stale or unreadable local image URLs are not replayed into later model requests.
- Temporary image access can be refreshed on retry to reduce expired URL failures.
- Image capability, route, and URL failures use clearer local error categories.

## Permission Recovery and Analytics

- Missing permissions now provide a clearer reason, target settings entry, return-to-app path, and one-time continuation for the original task.
- Location and Bluetooth permission recovery are split by capability scope.
- Firebase Analytics is isolated behind a provider contract and avoids user content or direct identifiers in events.

## Known Limitations

- This release was prepared as a GitHub pre-release and was not submitted to Google Play.
- Permission recovery had targeted test and limited-device coverage, while a broader Samsung, Xiaomi, and vivo matrix remained open.
- Top scenario regression, P0 failure sample gates, and at least three-device or three-Android-version compatibility coverage remained later stability gates.
- Custom model vision support is a user configuration flag and does not bypass provider capability limits.
